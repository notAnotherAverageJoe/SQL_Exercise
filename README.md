SQL Queries README
This repository contains a set of SQL queries along with explanations and use cases for each query. These queries are designed to retrieve specific information from a database table named analytics.

Queries Overview
Find the entire record for the app with an ID of 1880.

This query retrieves all fields for the app with the specified ID.
Find the ID and app name for all apps that were last updated on August 01, 2018.

This query selects the ID and app name for apps with the specified last update date.
Count the number of apps in each category.

This query provides a count of apps in each category.
Find the top 5 most-reviewed apps and the number of reviews for each.

This query retrieves the top 5 apps with the highest number of reviews.
Find the full record of the app with the most reviews and a rating greater than or equal to 4.8.

This query retrieves the app record with the highest number of reviews and a rating of 4.8 or higher.
Find the average rating for each category ordered by highest rated to lowest rated.

This query calculates the average rating for apps in each category.
Find the name, price, and rating of the most expensive app with a rating less than 3.

This query retrieves information for the most expensive app with a rating less than 3.
Find all records with a min install not exceeding 50, ordered by highest rated first.

This query retrieves records with a minimum installation count of 50 or less, ordered by rating in descending order.
Find the names of all apps rated less than 3 with at least 10000 reviews.

This query retrieves the names of apps with a rating less than 3 and at least 10000 reviews.
Find the top 10 most-reviewed apps that cost between 10 cents and a dollar.

This query retrieves the top 10 most-reviewed apps within a specified price range.
Find the most out-of-date app.

This query retrieves the app with the oldest last update date.
Find the most expensive app.

This query retrieves the app with the highest price.
Count all the reviews in the Google Play Store.

This query calculates the sum of all reviews across all apps.
Find all categories that have more than 300 apps.

This query retrieves categories with more than 300 apps.
Find the app with the highest proportion of reviews to min_installs among apps installed at least 100,000 times.

This query retrieves the app with the highest proportion of reviews to minimum installs among apps with at least 100,000 installs.
